
# 第十四章：项目中常用但未深入展开的 AWS 服务总览

> 本章目标：
>
> - 让学习者建立 **“AWS 服务全景认知”**
> - 知道在真实项目中，除了 EC2 / S3 / RDS / Lambda 之外，还常用哪些服务
> - 明确：**哪些需要“会用”，哪些只需要“知道存在”**
> - 为后续实际项目选型与深入学习提供导航

---

## 14.1 为什么需要这一章？

在真实项目中，你几乎 **不可能只用 5～6 个 AWS 服务**。

但好消息是：

> **你不需要精通所有 AWS 服务，只需要知道“什么时候该用谁”。**

这一章不追求细节操作，而是解决三件事：

- 这个服务是干什么的？
- 一般用在项目的哪一层？
- 我现在要不要学？

---

## 14.2 身份与安全类（Security & Identity）

### 14.2.1 AWS Secrets Manager

**作用：**

- 安全存储数据库密码、API Key、Token

**解决的问题：**

- 不把敏感信息写在代码或配置文件中

**典型使用：**

- EC2 / Lambda 从 Secrets Manager 读取 DB 密码

📌 建议程度：⭐⭐⭐⭐☆（生产常用）

---

### 14.2.2 AWS KMS（Key Management Service）

**作用：**

- 管理加密密钥

**典型场景：**

- S3 加密
- RDS 数据加密
- Secrets Manager 底层依赖

📌 建议程度：⭐⭐⭐☆☆（知道即可，通常“自动用”）

---

## 14.3 网络与访问控制类

### 14.3.1 Route 53

**作用：**

- AWS 的 DNS 服务

**典型用途：**

- 域名解析到 ALB / CloudFront
- 健康检查 + DNS 级故障切换

📌 建议程度：⭐⭐⭐⭐☆（几乎所有项目都会用）

---

### 14.3.2 AWS WAF

**作用：**

- Web 应用防火墙

**解决的问题：**

- 防 SQL 注入
- 防 XSS
- 防恶意爬虫

**位置：**

```text
用户 → WAF → CloudFront / ALB
```

📌 建议程度：⭐⭐⭐☆☆（对外系统常见）

---

## 14.4 消息与解耦类（非常重要）

### 14.4.1 SQS（消息队列）

**作用：**

- 异步解耦系统

**典型场景：**

- 下单 → 异步发邮件
- 高并发削峰

📌 关键词：
> 解耦 / 异步 / 削峰

📌 建议程度：⭐⭐⭐⭐⭐（真实项目非常常见）

---

### 14.4.2 SNS（通知服务）

**作用：**

- 发布 / 订阅消息
- 发送邮件、短信、Webhook

**常见组合：**

- CloudWatch Alarm → SNS → 邮件

📌 建议程度：⭐⭐⭐⭐☆

---

## 14.5 数据与分析类

### 14.5.1 DynamoDB

**作用：**

- NoSQL 数据库
- 高并发、低延迟

**常见搭配：**

- Lambda + DynamoDB

📌 不适合：

- 复杂 SQL 查询

📌 建议程度：⭐⭐⭐⭐☆（Serverless 常用）

---

### 14.5.2 Athena

**作用：**

- 直接用 SQL 查询 S3 中的数据

**典型用途：**

- 日志分析
- 报表查询

📌 建议程度：⭐⭐⭐☆☆（数据分析场景）

---

## 14.6 DevOps 与自动化类

### 14.6.1 IAM + CI/CD（Code 系列 / GitHub Actions）

**作用：**

- 自动部署
- 权限隔离

**典型流程：**

```text
Git Push → CI/CD → EC2 / Lambda
```

📌 建议程度：⭐⭐⭐⭐⭐（工程师必备）

---

### 14.6.2 CloudFormation / Terraform

**作用：**

- 基础设施即代码（IaC）

**解决的问题：**

- 环境可复现
- 自动化创建 AWS 资源

📌 建议程度：⭐⭐⭐⭐☆（中级工程师必学）

---

## 14.7 运维与治理类

### 14.7.1 AWS Config

**作用：**

- 资源配置审计
- 合规检查

📌 多用于：

- 金融 / 大型企业

---

### 14.7.2 AWS Cost Explorer

**作用：**

- 成本分析
- 找出“钱花在哪”

📌 建议程度：⭐⭐⭐⭐☆（非常实用）

---

## 14.8 一张“项目常用 AWS 服务地图”

```text
前端：
  S3 / CloudFront / Route53 / WAF

后端：
  ALB / EC2 / Lambda / API Gateway

数据：
  RDS / DynamoDB / S3 / Athena

解耦：
  SQS / SNS

安全：
  IAM / Secrets Manager / KMS

运维：
  CloudWatch / Config / Cost Explorer

自动化：
  CI/CD / CloudFormation / Terraform
```

---

## 14.9 学习建议（非常重要）

### 新手阶段

- 会用：EC2 / S3 / RDS / Lambda / ALB / CloudWatch
- 知道：SQS / SNS / Route53 / Secrets Manager

### 中级阶段

- 深入：SQS / DynamoDB / CI/CD / IaC

---

## 14.10 本章小结

- AWS 服务很多，但常用的其实有限
- 项目中最重要的是：**选对服务，而不是用得多**
- 你已经掌握了 80% 项目的核心 AWS 能力
- 剩下的是按需补充，而不是一次学完
